# Архитектура RemOnline

## Обзор

Бэкенд построен по принципам **Clean Architecture**: домен в центре, зависимости направлены внутрь. Слои:

- **Entities** — доменные сущности и сервисы (Order, Client, Employee и т.д.).
- **Application** — сценарии использования: command handlers, порты (интерфейсы), ошибки.
- **Infrastructure** — реализация портов: SQLAlchemy-адаптеры, миграции Alembic, Keycloak.
- **Presentation** — REST API (FastAPI), Pydantic-схемы, валидаторы.
- **Config** — настройки, DI (Dishka), логирование, маппинг таблиц.

## Слои и зависимости

```
presentation → application → entities
      ↓              ↓
   config         infra (адаптеры реализуют application ports)
```

- Роутеры вызывают command handlers (интеракторы), передают DTO и текущего сотрудника.
- Handlers используют порты (OrderReader, Transaction, EntitySaver и т.д.) и доменные сервисы (OrderService и т.д.).
- Адаптеры (infra) реализуют порты и работают с БД/Keycloak.

## Ключевые порты

- **Transaction** — commit, flush, rollback.
- **EntitySaver** — add_one, delete (для сущностей).
- **OrderReader, ClientReader, EmployeeReader, ...** — чтение сущностей по id/uuid/все.
- **AdminManager** (Keycloak) — регистрация пользователя, выдача токенов.

## Роли и доступ

- **SUPERVISOR** — полный доступ; единственный, кто может вводить реквизиты организации и редактировать супервизора.
- **ADMIN** — заказы, сотрудники (кроме редактирования супервизора), запчасти, статистика.
- **MANAGER** — создание/редактирование заказов, просмотр.
- **MASTER** — работы и запчасти по заказам, ограниченная смена статусов.

Проверка прав выполняется в presentation (RoleChecker) и в command handlers (например, запрет админу менять супервизора).

## Заказы

- Создание заказа: один агрегирующий endpoint (клиент + устройство + заказ) или пошагово.
- Цена заказа и допустимые переходы статусов считаются на бэкенде и отдаются в ответе чтения заказа (calculated_price, allowed_statuses).
- Печать: акт выполненных работ и квитанция о сдаче — HTML, в шаблоны подставляются реквизиты организации (singleton).

## Организация (реквизиты)

- Одна запись в таблице organizations (singleton_key = 1).
- CRUD через REST; доступ только у супервизора. Используется в шаблонах квитанции и акта.
